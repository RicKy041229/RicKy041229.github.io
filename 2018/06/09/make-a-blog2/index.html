用 Hexo+NexT 搭建个人博客 (二)

Posted by RicKy on 2018-06-09

I’m Back.

弃坑一周, 我又回来写教程了.

概要

这一篇写的是把自己的域名绑定到 Blog, 就是在访问自己的域名时解析到 yourid.github.io (CNAME). 其次, 就是如何在自己的 Blog 写下第一篇文章.

绑定自定义域名

创建 CNAME 文件

在你的 Blog 文件夹的 source 文件夹下, 新建一个名为 CNAME 的文件, 不要有后缀. (建个 .txt 的文本文档去掉后缀就好了)

新建好后, 右键 Edit with Notepad++ 打开, 在第一行输入你的域名, Ctrl+S 保存.

这样, 这个步骤就完成了一半了.

GitHub 仓库设置

打开 GitHub 仓库, 转到设置, 在 GitHub Pages 一栏找到 Custom domain , 把自己的域名填到框里, SAVE 保存即可.

配置域名解析

这里我推荐使用 Cloudflare, 因为它可以提供 HTTPS 和免费的 SSL 证书, 打开网站, 注册登录不用我详细介绍了, 能有一点英语基础的人都可以做到.

登录后会转到个人的主页, 点击 + Add a Site 来添加一个域名, 输入域名后点确认, 一路下一步, Select a Plan 那里选 FREE, 然后点确认, 弹出的框中点 Purchase.

之后会转到下面这样一个页面:

把这些都删掉, 在 Name 那一栏填你的域名; IPv4 address 如何获取, 我来说一下:

获取 IPv4 address

打开 cmd, 输入 ping yourid.github.io , 所得出的 IP 地址 即是你的 IPv4 address. 如图:

继续

把 IPv4 address 填好后, 点击 Add Record.即可添加一条解析.

同样的方法, Name 那一栏填 www, IPv4 address 还是那个, 再建立一条.

最后点击 Continue 即可.

进入 Overview, 域名那一栏的黑体字就是你的解析 DNS, 把它们填到域名服务商那里, 之后等待解析生效.

配置 Page Rules

在顶栏找到 Page Rules.

点击 Create Page Rule, 创建两条, 配置按下图填写:

注意: 域名要填自己的!

大功告成!

现在, 你只需要等待解析生效, 你就可以通过你自己的域名访问博客了.

年轻人的第一篇博文

新建 Markdown 文档

在你的 Hexo 文件夹中输入 hexo new title 即可创建一个文档, title 即是文档名.

打开文档并写作

文档生成在 blog/source/ 下.

这里打开 .md 文档我推荐使用 Typora, 打开后, 会看到如下内容:

没有特殊需要的话只改标题就可以, 写作并保存后, 使用 hexo clean && hexo g && hexo d 命令即可部署到 GitHub, 耐心等待几分钟后, 文章便呈现在了博客上.

The End.

又要摸了.

和之前一样, 如果有纰漏, 欢迎联系我修改, 第三部分准备写 配置 NexT 主题 以及 评论系统的添加 .
